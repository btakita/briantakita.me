## Tech Stack

### Solidjs provides benefits over other isomorphic component frameworks *(e.g. React)*:

<ul>
	<li>A more comprehensible API & more natural integration with the DOM API</li>
	<li>
		{[
			'Reactive Signals — ',
			<A_target_blank href="https://dev.to/this-is-learning/react-vs-signals-10-years-later-3k71">
				cleaner component state management
			</A_target_blank>
		]}
	</li>
	<li>
		{[
			'Smaller payload ',
			<em>
				(from
				<a href="https://dev.to/ryansolid/introducing-the-solidjs-ui-library-4mck"> Introducing the SolidJS UI Library</a>
				)
			</em>,
			<img width="50%" src="/assets/images/web-framework-realworld-app-sizes.png" alt="Web framework Realword App Sizes"/>
		]}
	</li>
	<li>
		<A_target_blank children="Fast" href="https://krausest.github.io/js-framework-benchmark/2023/table_chrome_118.0.5993.70.html"/>
	</li>
	<li>Less code needed for components</li>
	<li>JSX — function components facilitate composition *(compared to single file components with Vue or Svelte)*</li>
</ul>
